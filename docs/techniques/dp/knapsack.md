# Knapsack

## 介紹

背包問題是一種 **DP 問題**，若題目涉及固定容量填入一些項目，就算是背包問題的一種。

## 常見模型

### 0/1 背包

最基本的背包問題就是 0/1 背包，代表每種東西只有**取或不取**兩種可能。

::: info 例題

有 $n$ 個東西，具有重量 $w_i$ 與價值 $v_i$，現在你有一個容量 $W$ 的背包，請在 $\sum_{\text{chose}} w \le W$ 的情況下選一些東西使 $\sum_{\text{chose}} v$ 最大。

:::

dp 設計可以直覺想到：`dp[i][j]` 代表處理第 `i` 項物品且總重為 `j` 時，$\sum_{\text{chose}} v$  的最大值。

這時，每項物品可選可不選，轉移就會是：

- 若選第 `i` 項，`chmax(dp[i + 1][j + w[i]], dp[i][j] + v[i])`。
- 若不選第 `i` 項，`chmax(dp[i + 1][j], dp[i][j])`。

（註：`chmax(x, v)` 代表 `x = max(x, v)`）

這時的時間複雜度為 $O(nW)$

### 有限背包

若放入背包的物件有重複，就被稱為**有限背包**。

::: info 例題

有 $n$ 種東西，每種東西有 $c_i$ 個，並具有重量 $w$ 與價值 $v$，現在你有一個容量 $W$ 的背包，請在 $\sum_{\text{chose}} w \le W$ 的情況下選一些東西使 $\sum_{\text{chose}} v$ 最大。

:::

若使用 0/1 背包的精神，可以把這些重複的物品全部當成一個獨立的物品，這時的時間複雜度為 $O(W \sum c)$

#### 二進位拆包優化

當 $\sum c$ 達到 $10^5, 10^9, 10^{18}$ 量級，直接拆物品的方法就會超時。

這時，可以想到一個經典問題：

::: info 例題

想要湊出 $1 \sim 1023$ 之中的所有錢，至少需要幾個硬幣？

:::

可以想到把物品拆成 $2^0, 2^1, 2^2 \cdots$，不就可以把拆出的物品量從 $\sum c$ 變成 $\log \sum c$ 嗎？

但是，若直接硬拆，有可能讓總物品個數超出 $c_i$。

這時，可以想到經典的 $13 = 1 + 2 + 4 + 6$，只要把最後一個物品拆成：

$c_i - \max\{x \mid 2^x \le c_i\}$

就可以保證當全選所有拆出的物品時，總和必定是 $c_i$。
