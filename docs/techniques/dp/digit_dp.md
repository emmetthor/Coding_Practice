# Digit dp 數位 dp

## 這個 skill 解決什麼問題？

**數字極大**（$n \ge 9 \times 10^{18}$）的計數問題。

當然數字小的時候也可以用 digit dp 解決麻煩的條件。

## 使用時機

**數字超級大**，需要對每一位數做討論的題目。

## 核心想法

通常 digit dp 狀態由三個基本狀態組成：

| 狀態 | 意義 |
| --- | --- |
| `pos` | 目前處理到第幾位 |
| `tight` | 紀錄此位是否達到上限 |
| `started` | 紀錄數字是否開始，畢竟前綴零不算數字之一 |

而在搭配題目所需的其他狀態輔助，就可以設計出好的 digit dp。

通常轉移會使用 Memorized Search，畢竟狀態之間的轉移十分複雜。

::: info `tight` 是啥？為何要記錄這個？

`tight` 是紀錄此位是否達到上限，以上限為 $114514$ 為例，若第一位選擇 $1$，則第二位最大就是 $1$；但若第一位選擇 $0$，則第二位就可以選擇 $9$，因此要記錄 `tight` 來處理這種情況。

:::

## 思考流程

通常使用 digit dp 時，想法很簡單，直接以例題解釋。

::: info 例題

請求所有數字介於 $1 \sim N$，每一位數的乘積是 $M$ 的倍數的個數。

（$1 \le N \le 10^{500}$，$1 \le M \le 500$）

:::

1. 發現數字極大（$10^{500}$），必須使用 digit dp。
2. 發現題目需求是 $M$ 倍數，可以轉換成：對 $M$ 取餘的餘數為 $0$。
3. 於是可以將對 $M$ 取餘的餘數塞進狀態裡，因此狀態是：

::: tip 狀態設計

`dp[pos][tight][started][mod]` 代表在處理第 `pos` 位時，有無達到上限（`tight`），是否開始產生數字（`start`），且這時從頭到這裡的每位數乘積對 $M$ 取餘的餘數是 `mod`，有幾個數字滿足所有條件。

:::
