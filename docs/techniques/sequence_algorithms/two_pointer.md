# Two Pointer 雙指標

## 這個 skill 解決什麼問題？

已排序之**序列計數、極值問題**。滑動窗口問題。

## 使用時機

和序列中的**配對數據、連續子區間、滑動窗口**有關的問題皆可先排序在套用雙指標。

## 核心想法

有**兩個指標**分別指向前後，每次**移動指標時增量更新答案**。

## 常見模型

### 合併遞增序列

在合併排序時，當兩個已排序的序列要匯成一個排序序列，可以讓兩個指標分別**在兩個序列開頭**，**每次比較哪個較小就加入最終序列裡**。

```cpp
vector<int> a(n), b(m);
vector<int> ans;
for (int i = 0, j = 0; i < n; i++)
{
    while (j < m && a[i] > b[j]) ans.push_back(b[j++]);
    ans.push_back(a[i]);
}
```

### 單調序列的雙指標

在一個已排序的序列中，可以使用雙指標把一些 $O(n^2)$ 的計數透過排序的性質壓縮成 $O(n \log n) + O(n)$。

#### 同向指標問題

同向指標，代表兩個指標朝向同一方向移動且永不回頭。通常此雙指標每次移動指標時的問題核心是：

> 若 $l$ 固定，那 $r$ **最遠 / 最近可以放哪裡**？

因此同向雙指標常與**區間問題，滑動窗口問題**相關。

#### 反向指標問題

反向指標，代表兩個指標從左右端面對面前進。通常此雙指標每次移動指標時的問題核心是：

> 若 $l$ 固定，那 $r$ 所在的項目與 $l$ **配對**的結果對於**題目要求的太大還是太小**？

如果答案是「太大」，那往往是 `r--`；反之則是 `l++`。

若所求問題的算式有互補關係，也就是一方愈大，另一方必須愈小，就屬於反向指標問題的範疇。

> **[例題]** 請求出有幾組二元組 $(x, y)$ 可以使 $a_x + a_y = k$。
>
> 1. 由於當 $a_x$ 愈大，$a_y$ 必須愈小才能滿足總合為 $k$，因此適用反向指標。
> 2. 當每次 $a_l$ 與 $a_r$ 配對時，如果 $a_l + a_r < k$，就代表 $\{a_l + a_t < k\mid t \in [l + 1, r] \}$，因此必須使 $l + 1$；反之則代表 $\{a_t + a_r > k \mid t \in [l, r - 1]\}$，因此必須使 $r - 1$。
> 3. 當 $a_l + a_r = k$ 時，可以記錄答案。答案增加為 $\#\{a_t = a_l \mid t \in [l, r]\} \times \#\{a_t = a_r \mid t \in [l, r]\}$。

### 滑動窗口

滑動窗口是以 `l`, `r` 維護連續區間的技術，如果區間性質可以用 `+ 右端` 與 `- 左端` 來維護，則可以用雙指標把時間複雜度從 $O(n^2)$ 降到 $O(n)$。

::: info [例題] 對一個序列 $\in [1, \infty)$，請求出最長的連續子序列使總和 $\le k$。

1. 維護 `l` 與 `r` 代表目前觀察的序列的左右界。
2. 發現若右界每次往前一格，那左界最好的位置就是第一個使總和 $\le k$ 的位置。
3. 發現左界的最好位置必定會一直往右走。由於序列皆由正數組成，因此當右界擴張使總和增加，左界必須往右走來使總和 $\le k$，因此雙指標成立。

```cpp
vector<int> a(n);
int sum = 0;
int ans = 0;

for (int l = 0, r = 0; r < n;)
{
    // 若總和太大，就限縮左界
    while (!(sum <= k))
    {
        sum -= a[l++];
    }

    // 計算最大連續子區間長度
    ans = max(ans, r - l);

    // 擴張右界
    sum += a[r++];
}
```

:::

## 常見錯誤

- 未分清楚應使用的指標型態。
- 未**先分析**擴張**右端點**與擴張**左端點**的寫法優劣，就直接開始炸雙指標。
