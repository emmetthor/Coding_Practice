# DP Introduce

## DP 介紹

DP 是一種將**子問題的答案**紀錄下來以**解決大問題**的技巧，一個合理的 DP 狀態應符合：

1. 最佳子結構：大問題的最佳解必有部分在其**子問題的最佳解中**。
2. 將轉移與狀態化成圖後 (點：狀態；邊：轉移)，必須是 Directed Acyclic Graph (**DAG**, 有向無環圖)。

另外 DP 設計若滿足**子問題重疊**，才會顯示出 DP 的強大，反之則是普通分治問題。

> 若 DP 狀態有環出現，會使狀態更新時無法從現有的狀態得知，那其餘狀態更新時就必定錯。

![非 DAG DP 示意圖](../../../img/dp_introduce_cycle.png)

有另一種 DP 稱為 **Memorize Search**，是在遞迴過程中**記憶之前遞迴過的內容**。

## 狀態與時間複雜度簡寫

| DP 模型 | 狀態 | 轉移 | 時間複雜度 |
| - | - | - | - |
| 1D0D | 一維 | 常數 | $O(n)$ |
| 1D1D | 一維 | $O(n)$ | $O(n^2)$ |
| 2D0D | 二維 | 常數 | $O(n^2)$ |
| 2D1D | 二維 | $O(n)$ | $O(n^3)$ |
