# 實作規則 Implementation Rule

## 這個 skill 解決什麼問題？

當解題者似乎想到解法，想要開始實作時，卻發現其實**想法根本錯誤**，或是**實作太複雜**而導致解題失敗。

## 使用時機

所有需要**想 -> 實作 -> 抓錯**的題目。

## 核心想法

**用範例測資驗證 -> 找反例證非 -> 簡化模型 -> 寫下程式碼架構 -> 實作。**

## 思考流程

1. 以範例測資驗證想法正確性。
2. 盡可能 **[找反例]()** 去證明想法錯誤。
3. 盡可能簡化想法的模型，讓解題者清楚知道程式碼的實作架構。
4. 若模型仍然複雜，就需要將架構寫下來。
5. 可以開始實作了 :)。
